# Thesis Academic Revision Skill

面向中文本科论文的学术写作修订 skill。它的目标不是“洗稿”或“绕检测”，而是优先把论文里最容易露出机器味的几类问题改掉：断句不自然、标点乱用、形容词和程度副词过多。然后再处理空泛学术腔、模板连接词和大而空的总结句。

这个 skill 的规则，来自两版真实本科论文的对比总结。后面我把重点收得更贴近真实修改手法，核心规律是：

- 先修正断句。很多“AI 味”不是词本身，而是句子断得很怪。
- 优先处理句号、破折号、冒号和逗号的异常用法。
- 砍掉莫名其妙的形容词、评价词和程度副词。
- 再压缩模板连接词，比如 `因此`、`首先/其次/最后`、`从而`、`不仅……而且……`。
- 最后把抽象判断改成直接陈述，补足方法、系统设计和实验分析中的约束、步骤、模块、指标和结果。

## 适用场景

适合：

- 中文本科毕业论文
- 课程论文
- 开题后形成的章节草稿
- 摘要、引言、综述、方法、实验分析、结论、致谢

尤其适合：

- 计算机类
- 信息安全类
- 软件工程类
- 有“系统实现 + 实验验证”结构的工科论文

也可以泛化到其他中文本科论文，但专业术语和章节重点需要按学科微调。

## 不适合做的事

这个 skill 不会：

- 伪造实验结果
- 编造文献
- 代填不存在的数据
- 篡改原结论
- 承诺绕过任何检测器

如果原文缺证据，它会明确提示“需要作者补充”，而不是替作者瞎编。

## 推荐使用方式

不要整篇一把梭。更稳的方式是按章节或小节输入。

推荐粒度：

1. 摘要单独处理
2. 引言按 3-5 段处理
3. 文献综述按一个主题块处理
4. 方法与系统设计按一个模块或一个小节处理
5. 实验分析按一个图表组或一个结果段处理
6. 结论与致谢分别处理

## 调用示例

### 1. 处理引言

```text
请使用 thesis-academic-revision-zh skill 修订下面这段本科论文引言。
要求保留引用编号，减少空泛背景铺垫，把问题定义写得更直接。
```

### 2. 处理方法段

```text
请用 thesis-academic-revision-zh skill 修改这一节方法设计。
重点补清楚输入、处理、模块约束和输出，不要只是换同义词。
```

### 3. 处理实验分析

```text
请用 thesis-academic-revision-zh skill 修订这段实验分析。
把“效果较好、模型有效”改成更具体的指标和现象；如果原文没给数据，就标出需要补充的位置。
```

## 输出形式

skill 默认输出三部分：

1. `修订后的正文`
2. `本次修改摘要`
3. `仍需作者补充的信息`

这样你能直接看到：

- 改完的正文能不能粘回去
- 它这次具体动了哪些刀
- 还有哪些地方必须你自己补资料

## 使用建议

- 如果一段里句号断得很碎、破折号频繁插入、冒号乱用，通常就是第一优先级修订对象。
- 如果一段全是“显著、明显、较强、良好、稳定、高效、进一步”，通常说明修饰语过厚。
- 如果一段里“因此”“首先”“从而”“能够”很多，再继续压连接词和能力词。
- 方法段最怕写成概念说明，实验段最怕写成抽象赞美，结论最怕写成通用鸡汤尾句。

## 你会看到的典型改法

- 把硬拆开的短句重新合并，或者把该拆的长句拆干净
- 把没有必要的破折号、冒号改回正常句法
- 把“显著、明显、较强、良好、稳定、高效、进一步”这类词删掉一批
- “具有重要意义”改成“具体解决了什么问题”
- “能够提升泛化能力”改成“在未知样本上的表现更稳定”
- “系统采用轻量化路线”改成“为什么选这个框架、模块怎么拆、训练和推理如何共用特征定义”
- “实验结果表明效果较好”改成“在哪个数据集、用什么指标、相对谁更好”

## 维护方向

如果后面你想继续迭代这个 skill，可以优先补两类东西：

1. 更多学科特化规则
   比如经管、法学、教育学、文学类的常见空泛写法。

2. 更多真实 before/after 样例
   这比继续堆抽象规则更有用，模型学得也更快。
