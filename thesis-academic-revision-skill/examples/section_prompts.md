# Section Prompt Examples

下面这些提示词是给使用这个 skill 时参考的。推荐按章节或小节调用，不要整篇一次性输入。

## 1. 摘要

```text
请使用 thesis-academic-revision-zh skill 修订下面这段本科论文摘要。
要求：
1. 保留原意和引用信息；
2. 先修正断句和标点异常；
3. 删除多余形容词和程度副词；
4. 明确研究对象、方法、结果；
5. 如果结果部分缺少具体指标，请在“仍需作者补充的信息”里标出来。
```

## 2. 引言

```text
请使用 thesis-academic-revision-zh skill 修订下面这段引言。
重点：
1. 先处理奇怪断句、破折号和冒号；
2. 删除“长期以来、随着……发展、具有重要意义”这类空泛背景；
3. 保留研究背景，但把问题定义写得更直接；
4. 保留原引用编号；
5. 不要写成口语化表达。
```

## 3. 文献综述

```text
请用 thesis-academic-revision-zh skill 修改这一段文献综述。
要求把“作者名单式罗列”压缩成“代表工作 + 主要贡献/局限”，
减少“验证了有效性、具有重要价值”这类套话，
但不要编造论文中没有写出的局限。
```

## 4. 方法设计

```text
请用 thesis-academic-revision-zh skill 修订这一节方法设计。
重点补清楚：
- 输入是什么
- 中间处理步骤是什么
- 关键约束是什么
- 输出是什么

不要只做同义词替换。先删奇怪断句、形容词和副词，再重构句子和段落逻辑。
```

## 5. 系统实现

```text
请使用 thesis-academic-revision-zh skill 修改这段系统实现说明。
要求把“系统采用某技术路线、具有良好扩展性”之类空话改成更具体的实现描述，
写清模块划分、数据流和这样设计的原因。
```

## 6. 实验分析

```text
请用 thesis-academic-revision-zh skill 修订这段实验分析。
要求：
1. 先处理断句和标点异常；
2. 把“模型有效、效果较好”以及“显著、明显、较强”等程度词改成更具体的指标和现象；
3. 图表说明按“观察 -> 解释 -> 结论”的顺序来写；
4. 如果缺少数据，请明确标记需要作者补充，而不是代写结果。
```

## 7. 结论

```text
请用 thesis-academic-revision-zh skill 修改这段结论。
要求：
1. 只总结正文里已经完成的内容；
2. 删除“奠定基础、广阔前景、重要意义”这类通用尾句；
3. 如果需要提局限，语气保持正式克制。
```

## 8. 致谢

```text
请用 thesis-academic-revision-zh skill 修订这段致谢。
要求保留感谢对象，但减少模板化套话，
尽量写清楚老师、同学或家人分别提供了什么帮助，
语气自然一些，但不要过度口语化。
```

## 9. 混合小节处理

```text
请使用 thesis-academic-revision-zh skill 修订下面这一小节。
先判断每段属于背景、综述、方法还是分析，
再按段落类型分别处理。
输出时请提供：
1. 修订后的正文
2. 本次修改摘要
3. 仍需作者补充的信息
```

## 10. 高风险段落抢修

```text
请用 thesis-academic-revision-zh skill 优先处理下面这些高风险段落。
这些段落里句号断得很碎，破折号和冒号也用得很乱，同时还有很多“显著、明显、较强、进一步”这类词。
请重点：
1. 修正断句和标点；
2. 删除多余形容词和程度副词；
3. 再降低模板连接词密度；
4. 把抽象判断改成更具体的事实表达；
5. 保持正式论文语体。
```
